7. Variance Interpolation — Worked Example (BTC 30D)

Bracketing expiries: 27MAR26 (DTE=20.94, ATM IV=52.88%) → 24APR26 (DTE=48.94, ATM IV=50.81%)

Step 1: Convert to years: t₁ = 20.94/365.25 = 0.0573, t₂ = 48.94/365.25 = 0.1340, t_target = 30/365.25 = 0.0821

Step 2: Convert IV to decimal: σ₁ = 0.5288, σ₂ = 0.5081

Step 3: Compute variances: v₁ = σ₁²×t₁ = 0.01603, v₂ = σ₂²×t₂ = 0.03459

Step 4: Interpolate: v_target = 0.01603 + (0.03459−0.01603) × (0.0821−0.0573)/(0.1340−0.0573) = 0.02204

Step 5: Invert: σ_30d = √(0.02204 / 0.0821) × 100 = 51.80%

Dashboard shows: 51.90% — diff of 0.10 vol points from live mark_iv drift between my REST fetch and the dashboard's WebSocket feed. ✅ Confirmed correct.

8. Source of Residual Differences

All observed differences (0.01–0.32 vol points) are explained by:

1. Timing mismatch: My verification used Deribit REST API snapshots fetched sequentially over ~5 minutes. The dashboard uses WebSocket streaming data updated in real-time. Crypto vol surfaces move continuously.

2. Delta source: For RR calculations, I used REST ticker greeks (polled). The dashboard uses WebSocket ticker.100ms greeks (streaming). Delta values shift as spot moves, causing small RR differences.

3. Spot price drift: The dashboard's WebSocket spot feed and my REST index_price fetch can differ by ~$100-200 during active markets. This affects which strikes are "nearest ATM" and shifts interpolation weights.

None of these represent calculation errors. They are inherent to comparing two independent snapshots of a live market taken seconds apart.
